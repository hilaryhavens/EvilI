========================================================================
1. PROJECT
========================================================================
EvilI: a static website (GitHub Pages) that analyzes long-eighteenth-
century British texts entirely in the browser and reports:
  - whether the text is first-person narration
  - narrator reliability (0–100)
  - narrator morality, two scores: deeds vs. self-presentation
    (−100 immoral .. +100 virtuous), whose gap doubles as the
    deed/word-discrepancy unreliability signal
Inputs: paste, .txt upload (batch), TEI/XML, bundled sample corpus.
Outputs: evidence-highlighted scores, charts (narrative arc, scatter),
CSV/JSON export, PNG/SVG chart export, cross-text comparison.
No server, no tracking, no data retention (structural: analysis never
leaves the browser). Audience: researchers (rigor + documentation) AND
students/public (friendly UI).

Key design decisions (full detail in the two docs below):
  - In-browser NLP heuristics (no LLM). Vite + TypeScript + wink-nlp,
    analysis in a Web Worker; hand-rolled SVG charts (not Chart.js, so
    SVG export works cleanly).
  - Reliability signals: S1 hedging (weight 1 — deliberately low),
    S2 self-justification/reader appeals (2), S3 deed/word gap (2),
    S4 contradiction/retraction (2).
  - Morality counted only for first-person agency ("I stole" counts,
    "he stole" / "I never stole" don't).
  - OCR remediation for ECCO texts (long-s-as-f etc.), dictionary-gated:
    repair only if unknown token -> known word; quality report + UI
    warning above 5% unknown tokens.
  - Lexicons are editable JSON data files — seed sets shipped (~20–30
    entries each); expanding to the spec's 50–150 per category is
    editorial curation work for the project owner.
